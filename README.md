# Структура адаптации

Сборка объектов сервера и бекенда шаблонов осуществляется с помощью пакета wsjs

Файлы .gitKeep можно удалить после инициализации других файлов в папке, они созданы исключительно для сохранения структуры в гит

## Объекты сервера (server-objects)

Разработка ведется в директории ``server-objects/src``.

Можно создавать общие утилиты в корне директории для использования во всех заданиях

Модули разбиты на папки, задания необходимо выполнять в соответствующих папках:

| Модуль  | Папка |  Комментарий    |
|-------|-----|-------|
| Агенты | agents  | - |
| Уведомления   | notifications  | -   |
| Выборки   | remote-collections  | -   |
| Выгрузки   | discharge  | Можно не заносить код в репозиторий, но показать работоспособность наставнику   |
| Настраиваемые отчеты   | reports  | Можно не заносить код в репозиторий, но показать работоспособность наставнику   |
| Системные события   | system-events  | -   |
| Печатные формы   | print-forms  | Шаблон формы лежит в папке. В рамках задачи можно не делать UI а написать только бекенд (веб-бекенд или удаленное действие)  |
| API   | api  | -   |
| Кастомные каталоги   | custom-catalogs  | -   |
| LPE   | lpe  | -   |
| Удаленные действия   | remote-actions  | -   |

## Веб-шаблоны (web-templates)

### Выполнение заданий модуля "Настраиваемые web-шаблоны"

- Все задания должны выполняться последовательно, новые отдельные папки под задания не создаются, а дорабатываются файлы, созданные с прошлого задания
- Фронтенд должен лежать в папке ``frontend/web-template-module``
- Бекенд должен лежать в папке ``backend/src/web-template-module``

### Выполнение заданий модуля "Процедура оценки"

- Фронтенд должен лежать в папке ``frontend/assesment-procedure``
- Бекенд должен лежать в папке ``backend/src/assesment-procedure``
